# YouTube Desktop App

YouTubeとYouTube Musicの体験を向上させるための包括的な管理デスクトップアプリケーション

## 概要

このアプリケーションは、YouTube Data API v3と直接統合し、チャンネル、プレイリスト、最新動画などを一元的に表示・管理します。

## 主な機能

### 統合ダッシュボード (ホームページ)
- **最新情報**: 登録チャンネルの最新動画を表示
- **登録チャンネル**: YouTubeで登録しているすべてのチャンネルを一覧表示
- **アーティスト**: 音楽関連のチャンネルを自動判別して表示
- **再生リスト**: 作成した再生リストを管理
- **YouTube Music プレイリスト**: 音楽関連の再生リストを自動判別して表示

### キャッシュシステム
- 取得したデータはMongoDBにキャッシュされ、2回目以降の表示を高速化
- バックグラウンドで定期的に差分を更新し、APIクォータ消費を最小限に抑制
- 起動時に全データを強制的に同期するオプション付き

### データ判定
- チャンネルやプレイリストが音楽関連かどうかを、含まれる動画のカテゴリ（カテゴリID: 10）に基づいて自動判定
- 50%以上の動画が音楽カテゴリの場合、そのチャンネル/プレイリストを音楽関連と判定

### 認証
- Google OAuth 2.0による安全な認証
- アクセストークンは自動的に管理され、必要に応じて再発行

## 技術スタック

- **Electron**: デスクトップアプリケーションフレームワーク
- **React**: UIフレームワーク
- **TypeScript**: 型安全な開発
- **MongoDB**: データキャッシュ用データベース
- **YouTube Data API v3**: YouTube データの取得

## セットアップ

### 前提条件

- Node.js (v18以上推奨)
- MongoDB (ローカルまたはMongoDB Atlas)
- Google Cloud Platform アカウント

### Google OAuth 2.0 設定

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. 新しいプロジェクトを作成
3. YouTube Data API v3を有効化
4. 認証情報を作成（OAuth 2.0 クライアントID）
5. 承認済みのリダイレクトURIに `http://localhost:3000/oauth2callback` を追加
6. クライアントIDとクライアントシークレットをメモ

### インストール

```bash
# 依存関係のインストール
npm install

# 環境変数の設定
cp .env.example .env
# .envファイルを編集してGoogle認証情報とMongoDB URIを設定
```

### 開発

```bash
# 開発モードで起動
npm run dev
```

### ビルド

```bash
# アプリケーションのビルド
npm run build

# ビルド後の起動
npm start
```

## 使い方

1. アプリケーションを起動
2. 「Googleでログイン」ボタンをクリック
3. ブラウザでGoogle認証を完了
4. ダッシュボードで各種データを閲覧

### 機能説明

- **ホームタブ**: 最新動画と各セクションの概要を表示
- **登録チャンネルタブ**: すべての登録チャンネルを表示
- **アーティストタブ**: 音楽関連チャンネルのみを表示
- **再生リストタブ**: すべての再生リストを表示
- **YouTube Musicプレイリストタブ**: 音楽関連の再生リストのみを表示
- **強制同期ボタン**: キャッシュをクリアして最新データを取得
- **キャッシュクリアボタン**: すべてのキャッシュデータを削除
